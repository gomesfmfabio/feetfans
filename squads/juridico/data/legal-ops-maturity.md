## 4 Estagios de Maturidade

| Estagio | Nome | Caracteristicas |
|---------|------|-----------------|
| 1 | **Ad Hoc** | Sem processos formais, reativo, dependente de individuos |
| 2 | **Basic** | Processos basicos documentados, alguma padronizacao |
| 3 | **Managed** | Processos consistentes, metricas definidas, melhoria continua |
| 4 | **Optimized** | Data-driven, automatizado, previsivel, inovacao constante |

## Assessment Checklist

Para cada area (1-12), avaliar:
- [ ] Estagio atual (1-4)
- [ ] Estagio desejado
- [ ] Gap identificado
- [ ] Prioridade (Alta/Media/Baixa)
- [ ] Acoes recomendadas
- [ ] Responsavel
- [ ] Timeline

## Uso no LegalOS

O @richard-susskind usa este framework via comando `*diagnose-maturity` para:
1. Avaliar o estagio atual em cada area
2. Identificar gaps prioritarios
3. Recomendar acoes de evolucao
4. Definir roadmap de maturidade
