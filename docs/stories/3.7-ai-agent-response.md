## Technical Notes

```typescript
async function generateAgentResponse(agentId: string, conversationId: string, creatorMessage: string) {
  // Fetch conversation history
  const { data: history } = await supabase
    .from('messages')
    .select('content, sender_id')
    .eq('conversation_id', conversationId)
    .order('created_at', { ascending: false })
    .limit(5);

  // Check if creator shared link
  const hasLink = /https?:\/\//.test(creatorMessage);

  const { data: creator } = await supabase
    .from('users')
    .select('subscription_status')
    .eq('id', creatorId)
    .single();

  let prompt = `${agent.personality_prompt}\n\nConversation history:\n${formatHistory(history)}\n\nCreator said: "${creatorMessage}"\n\nRespond naturally (<75 words).`;

  if (hasLink && creator.subscription_status === 'trial') {
    prompt += ' The creator mentioned a link but you cannot see it. Respond that you don\'t see any link.';
  } else if (hasLink) {
    prompt += ' The creator shared a link. Politely acknowledge but make an excuse not to click it now.';
  }

  // Call Claude API
  const message = await anthropic.messages.create({
    model: 'claude-3-haiku-20240307',
    max_tokens: 150,
    messages: [{ role: 'user', content: prompt }],
  });

  return message.content[0].text;
}
```

## Dev Agent Record

### Implementation Summary
Created AI agent automatic response system with conversation history context, link handling for trial/paid users, and purchase protection.

### Technical Decisions
- **Async response generation:** Don't block message POST, trigger AI response asynchronously
- **AI agent detection:** Check if recipient exists in ai_agents table
- **Conversation history:** Fetch last 5 messages for context
- **Link detection:** Regex check for URLs in creator message
- **Trial vs paid handling:** Different prompts based on subscription_status
- **Purchase protection:** Explicit instructions in prompt to never agree to purchases
- **Max tokens 150:** Supports up to ~75 words responses
- **Temperature 0.8:** Add variety to responses

### Features Implemented
**Response Generation:**
- generateAgentResponse(agentId, conversationId, creatorMessage, creatorId)
- Fetch agent personality and conversation history (last 5)
- Format history as "You: ..." / "Creator: ..." chronologically
- Detect links in creator message with regex

**Link Handling:**
- **Trial users:** Agent says "I don't see any link, can you describe it?"
- **Paid users:** Agent acknowledges but makes excuse (mobile, prefer chat, etc.)

**Purchase Protection:**
- Added to every prompt: "NEVER agree to purchase, NEVER ask for payment"
- Ensures agents stay in browsing/chatting role only

**Integration:**
- POST /api/messages endpoint detects AI recipient
- Determines recipient from conversation (creator_id vs consumer_id)
- Checks ai_agents table for recipient existence
- Triggers generateAgentResponse async (no blocking)
