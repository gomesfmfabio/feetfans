# Story 1.1: Repository Setup & CI/CD Pipeline

**Epic:** 1 - Foundation & Core Infrastructure
**Status:** Draft
**Assigned to:** @dev
**Estimated hours:** 4
**Dependencies:** None (first story in epic)

---

## Story Description

Set up the foundational monorepo structure for the FeetFans PWA marketplace project. This includes initializing a Turborepo-based monorepo with pnpm workspaces, configuring code quality tools (ESLint, Prettier, TypeScript), setting up pre-commit hooks, and establishing a CI/CD pipeline with GitHub Actions.

This is a **greenfield project** - there is no existing git repository. This story will initialize the git repository and establish the foundation for all future development.

---

## Acceptance Criteria

- [ ] **AC1:** Monorepo initialized with Turborepo
  - Directory structure: `/apps/web`, `/apps/api`, `/packages/shared`
  - `turbo.json` configured with build, lint, test, dev pipelines
  - `pnpm-workspace.yaml` configured to include all workspaces

- [ ] **AC2:** ESLint, Prettier, TypeScript configs with strict rules
  - Root-level ESLint config extends Next.js and TypeScript best practices
  - Prettier config with consistent formatting rules (semicolons, quotes, etc.)
  - TypeScript `tsconfig.json` with `strict: true` and path aliases
  - Pre-commit hooks configured with Husky + lint-staged to run linting/formatting

- [ ] **AC3:** GitHub Actions workflow for linting, type checking, testing
  - Workflow file at `.github/workflows/ci.yml`
  - Triggers on push to `main` branch and pull requests
  - Jobs: lint (ESLint), typecheck (tsc), test (npm test)
  - Uses pnpm with caching for faster builds

- [ ] **AC4:** Environment variable management documented
  - `.env.example` file with all required environment variables for dev/staging/prod
  - Variables include: `DATABASE_URL`, `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `ONFIDO_API_TOKEN`
  - `.gitignore` includes `.env`, `.env.local`, `.env.*.local`

- [ ] **AC5:** README with setup instructions
  - Prerequisites: Node.js 18+, pnpm, Git
  - Installation steps: clone repo, install dependencies, copy `.env.example` to `.env`
  - Development commands: `pnpm dev`, `pnpm build`, `pnpm lint`, `pnpm test`
  - Architecture overview: brief description of monorepo structure

## Technical Notes

### Turborepo Configuration

```json
// turbo.json
{
  "$schema": "https://turbo.build/schema.json",
  "pipeline": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": [".next/**", "dist/**"]
    },
    "lint": {},
    "test": {},
    "dev": {
      "cache": false,
      "persistent": true
    }
  }
}
```

### pnpm Workspace Configuration

```yaml
# pnpm-workspace.yaml
packages:
  - 'apps/*'
  - 'packages/*'
```

### GitHub Actions Workflow

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
        with:
          version: 8
      - uses: actions/setup-node@v4
        with:
          node-version: 18
          cache: 'pnpm'
      - run: pnpm install
      - run: pnpm lint

  typecheck:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
        with:
          version: 8
      - uses: actions/setup-node@v4
        with:
          node-version: 18
          cache: 'pnpm'
      - run: pnpm install
      - run: pnpm typecheck

  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
        with:
          version: 8
      - uses: actions/setup-node@v4
        with:
          node-version: 18
          cache: 'pnpm'
      - run: pnpm install
      - run: pnpm test
```

## Implementation Steps

1. **Initialize git repository** (since this is greenfield):
   ```bash
   git init
   git branch -M main
   ```

2. **Install Turborepo and create monorepo structure**:
   ```bash
   pnpm dlx create-turbo@latest
   # Configure with pnpm, create apps/web, apps/api, packages/shared
   ```

3. **Configure TypeScript with strict mode**:
   - Root `tsconfig.json` with `"strict": true`
   - Workspace-specific tsconfig files extend root

4. **Set up ESLint and Prettier**:
   ```bash
   pnpm add -D eslint prettier eslint-config-next eslint-config-prettier
   ```
   - Create `.eslintrc.json` and `.prettierrc`

5. **Configure pre-commit hooks**:
   ```bash
   pnpm add -D husky lint-staged
   npx husky install
   ```
   - Create `.husky/pre-commit` script
   - Configure `lint-staged` in `package.json`

6. **Create GitHub Actions workflow**:
   - Create `.github/workflows/ci.yml`
   - Configure lint, typecheck, test jobs

7. **Document environment variables**:
   - Create `.env.example` with all required variables
   - Update `.gitignore`

8. **Write README with setup instructions**

9. **Initial commit and verify CI/CD**:
   ```bash
   git add .
   git commit -m "feat: initialize monorepo with Turborepo, ESLint, Prettier, Husky, GitHub Actions"
   git push origin main
   ```

## Risks & Mitigations

**Risk:** GitHub Actions might fail on first run due to missing test files
**Mitigation:** Create placeholder test files in each workspace that pass by default

**Risk:** Husky pre-commit hooks might be too strict and slow down commits
**Mitigation:** Configure lint-staged to only check staged files, not entire codebase
