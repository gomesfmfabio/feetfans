## Technical Notes

```typescript
import Anthropic from '@anthropic-ai/sdk';

const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });

async function generateAgentMessage(agentId: string, creatorId: string) {
  // Fetch agent personality
  const { data: agent } = await supabase
    .from('ai_agents')
    .select('personality_prompt, nickname')
    .eq('id', agentId)
    .single();

  // Fetch creator's recent content
  const { data: content } = await supabase
    .from('content')
    .select('categories')
    .eq('creator_id', creatorId)
    .order('created_at', { ascending: false })
    .limit(5);

  const categories = content.flatMap(c => c.categories).join(', ');

  // Construct prompt
  const prompt = `${agent.personality_prompt}\n\nThe creator has posted content in these categories: ${categories}.\n\nWrite a brief, friendly opening message (max 50 words) expressing interest in their content. Do not reveal you are AI.`;

  // Call Claude Haiku
  const message = await anthropic.messages.create({
    model: 'claude-3-haiku-20240307',
    max_tokens: 100,
    messages: [{ role: 'user', content: prompt }],
  });

  // Log cost
  console.log(`Tokens used: ${message.usage.input_tokens + message.usage.output_tokens}`);

  return message.content[0].text;
}
```

## Dev Agent Record

### Implementation Summary
Created AI message generation system using Claude Haiku API with retry logic, rate limiting, cost tracking, and integration with cron job assignment processor.

### Technical Decisions
- **Claude Haiku model:** cost-effective for short messages (<100 tokens output)
- **Retry with exponential backoff:** 3 attempts with 1s, 2s, 4s delays
- **In-memory rate limiting:** Track message timestamps, max 10/minute
- **Temperature 0.8:** Add variety to agent responses
- **Token limit:** 100 max_tokens for <50 word responses
- **Cost tracking:** Log input/output tokens, warn if >500 total
- **Conversation context:** Fetch recent creator content categories
- **New creator handling:** Special prompt for creators without content yet

### Features Implemented
**Message Generation:**
- generateAgentMessage(agentId, creatorId, conversationId)
- Fetch agent personality_prompt and creator content categories
- Construct contextual prompt with personality + content interests
- Call Claude Haiku API with temperature 0.8 for variety
- Parse response and validate non-empty

**Error Handling:**
- 3 retry attempts with exponential backoff (1s → 2s → 4s)
- Detailed error logging with attempt count
- Graceful failure with error message return

**Rate Limiting:**
- In-memory timestamp tracking (messageTimestamps array)
- Check before each generation attempt
- getRateLimitStatus() for monitoring

**Cost Tracking:**
- Log input_tokens, output_tokens, total per message
- Warn if total > 500 (cost target)
- Console logs for monitoring

**Integration:**
- Updated cron job endpoint to call generateAgentMessage after conversation creation
- Fetch conversation_id from assignment
- Track message_sent vs error status
