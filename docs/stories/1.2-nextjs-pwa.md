## Story Description

Bootstrap the FeetFans marketplace progressive web application (PWA) using Next.js 14+ with App Router and TypeScript. Configure PWA capabilities with next-pwa plugin to enable offline support, installability, and native-like experience on mobile devices. Set up Tailwind CSS for mobile-first responsive design and create the initial landing page with branding.

## Technical Notes

### Next.js App Router Structure

```
apps/web/
├── app/
│   ├── layout.tsx          # Root layout with metadata
│   ├── page.tsx            # Landing page (/)
│   ├── globals.css         # Global styles + Tailwind
│   └── favicon.ico
├── public/
│   ├── manifest.json
│   ├── icons/
│   │   ├── icon-192x192.png
│   │   ├── icon-512x512.png
│   │   └── apple-touch-icon.png
│   └── sw.js               # Auto-generated by next-pwa
├── next.config.js
├── tailwind.config.js
└── postcss.config.js
```

### PWA Configuration (next.config.js)

```javascript
const withPWA = require('next-pwa')({
  dest: 'public',
  register: true,
  skipWaiting: true,
  disable: process.env.NODE_ENV === 'development',
});

module.exports = withPWA({
  reactStrictMode: true,
  swcMinify: true,
});
```

## Quality Gates

### Manual Quality Gate 1: Verify PWA is installable
**Steps:**
1. Run `pnpm --filter @feetfans/web dev`
2. Open http://localhost:3000 in Chrome
3. Check DevTools > Application > Manifest (should show FeetFans metadata)
4. Check DevTools > Application > Service Workers (should be registered)
5. Verify install button appears in browser address bar

**Expected:** Manifest loads correctly, service worker registers, install prompt appears

## Implementation Steps

1. **Install dependencies:**
   ```bash
   cd apps/web
   pnpm add next-pwa
   pnpm add -D tailwindcss postcss autoprefixer
   ```

2. **Initialize Tailwind CSS:**
   ```bash
   npx tailwindcss init -p
   ```

3. **Configure next.config.js with PWA wrapper**

4. **Create manifest.json in public/ directory**

5. **Create PWA icons:**
   - Generate 192x192 and 512x512 icons
   - Create apple-touch-icon.png (180x180)
   - Place in public/icons/

6. **Set up Tailwind CSS:**
   - Configure tailwind.config.js with content paths
   - Create app/globals.css with Tailwind directives

7. **Update app/layout.tsx:**
   - Add metadata (title, description, manifest link)
   - Include globals.css
   - Add responsive navigation structure

8. **Create landing page (app/page.tsx):**
   - FeetFans branding
   - "Coming Soon" message
   - Mobile-first responsive design

9. **Test PWA:**
   - Run dev server
   - Check manifest in DevTools
   - Verify service worker registration
   - Test install prompt

10. **Commit changes:**
    ```bash
    git add .
    git commit -m "feat: add Next.js PWA with Tailwind CSS [Story 1.2]"
    ```

## Risks & Mitigations

**Risk:** PWA might not work correctly in iOS Safari due to Apple's limited PWA support
**Mitigation:** Test on physical iOS device, ensure apple-touch-icon is configured, verify standalone mode works

**Risk:** Service worker caching might cause issues during development
**Mitigation:** Disable PWA in development mode via next-pwa config
