## Technical Notes

```sql
CREATE TABLE agent_assignments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  creator_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  agent_id UUID NOT NULL REFERENCES ai_agents(id),
  scheduled_for TIMESTAMPTZ NOT NULL,
  sent_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

```typescript
// Assign agents on creator signup
function assignAIAgents(creatorId: string) {
  const agents = await supabase.from('ai_agents').select('id').eq('is_active', true);

  const assignments = agents.map((agent, index) => ({
    creator_id: creatorId,
    agent_id: agent.id,
    scheduled_for: new Date(Date.now() + randomDays(0, 10) * 24 * 60 * 60 * 1000),
  }));

  await supabase.from('agent_assignments').insert(assignments);
}

// Cron job (runs hourly)
node-cron.schedule('0 * * * *', async () => {
  const due = await supabase
    .from('agent_assignments')
    .select('*')
    .lte('scheduled_for', new Date())
    .is('sent_at', null);

  for (const assignment of due) {
    await triggerAIMessage(assignment);
  }
});
```

## Dev Agent Record

### Implementation Summary
Created agent assignment system with automatic scheduling on creator verification, database triggers for auto-assignment, and hourly cron job to process due assignments.

### Technical Decisions
- **Trigger-based assignment:** Uses PostgreSQL trigger on users table when age_verified changes to true for creators
- **Function-based randomization:** `assign_ai_agents_to_creator()` SQL function generates random 0-10 day delays
- **Batch processing:** Cron job processes max 100 assignments per run to avoid timeout
- **Conversation creation:** Uses INSERT ... ON CONFLICT to get or create conversation
- **Error handling:** Individual assignment failures don't block batch processing

### Features Implemented
**Database:**
- agent_assignments table with unique constraint (creator_id, agent_id)
- Indexed on scheduled_for for fast cron queries
- Trigger function to auto-assign on age verification
- Batch processing function with error handling

**Cron Job:**
- GET endpoint with Bearer token auth (CRON_SECRET)
- Calls process_due_agent_assignments() SQL function
- Returns processed count, successes, failures
- 5-minute timeout for large batches
