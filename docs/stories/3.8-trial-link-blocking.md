## Technical Notes

```typescript
const URL_REGEX = /https?:\/\/[^\s]+/g;

function stripLinks(message: string, userStatus: string): string {
  if (userStatus === 'trial') {
    return message.replace(URL_REGEX, '[link removed]');
  }
  return message;
}

// Before passing to AI
const processedMessage = stripLinks(creatorMessage, creator.subscription_status);
```

## Dev Agent Record

### Implementation Summary
Created link blocking system for trial users, ensuring AI agents never see URLs shared by trial creators while allowing paid users to share freely.

### Technical Decisions
- **Regex-based detection:** `https?:\/\/[^\s]+/gi` matches all HTTP/HTTPS URLs
- **Replace with placeholder:** Links replaced with "[link removed]" text
- **Trial-only blocking:** Paid users (active/cancelled) can share links normally
- **Pre-processing:** Strip links before passing message to AI prompt
- **Banner already exists:** Chat screen already has trial banner mentioning links (Story 3.3)

### Features Implemented
**Link Blocker Utilities:**
- `stripLinksForTrial(message, userStatus)` - Main function
- `hasLinks(message)` - Boolean check for URL presence
- `countLinks(message)` - Count number of URLs in message

**Integration:**
- Updated generateAgentResponse to call stripLinksForTrial after fetching creator status
- Process message before constructing AI prompt
- Keep original message for link detection (hasLink variable)
- Pass processed message (links removed) to Claude API

**Behavior:**
- **Trial user sends link:** AI sees "[link removed]" and responds per AC5/AC6 logic (Story 3.7)
- **Paid user sends link:** AI sees full message with link, responds with excuse
- **No link:** Both trial and paid users' messages pass through unchanged
