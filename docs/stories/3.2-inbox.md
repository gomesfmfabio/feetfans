## Technical Notes

```typescript
const { data: conversations } = await supabase
  .from('conversations')
  .select(`
    id,
    creator:users!creator_id(nickname),
    consumer:users!consumer_id(nickname),
    messages(content, created_at)
  `)
  .or(`creator_id.eq.${userId},consumer_id.eq.${userId}`)
  .order('messages.created_at', { ascending: false })
  .limit(1, { foreignTable: 'messages' });
```

## Dev Agent Record

### Implementation Summary
- Created messages inbox page at /messages with conversations list
- Implemented real-time updates using Supabase Realtime subscriptions
- Displays conversation info: avatar, nickname, last message preview, timestamp, unread count
- Sorted conversations by most recent message first

### Technical Decisions
- Used Supabase Realtime for real-time inbox updates (subscribes to messages INSERT and conversations UPDATE)
- Fetch last message and unread count separately for each conversation for accurate data
- Map creator/consumer from array to object (Supabase joins return arrays)
- Determine "other user" dynamically based on current user's role in conversation
- Sort by lastMessage.created_at or conversation.updated_at as fallback
- Truncate message preview to 50 characters with ellipsis

### Features Implemented

**Conversation List:**
- Avatar with first letter of other user's nickname
- Conversation nickname (creator or consumer based on current user)
- Last message preview with "You: " prefix if sent by current user
- Relative timestamp (Just now, 5m ago, 2h ago, 3d ago, or date)
- Unread badge showing count (max 9+)
- Hover effect on list items
- Click to navigate to chat screen

**Last Message Logic:**
- Fetches most recent message per conversation
- Shows preview with sender indication
- Falls back to "No messages yet" if no messages
- Truncates long messages to 50 chars

**Unread Count:**
- Counts messages from other user with read_at = null
- Red badge with white text
- Shows "9+" for counts > 9
- Only displays if count > 0

**Real-Time Updates:**
- Subscribes to `messages` INSERT events
- Subscribes to `conversations` UPDATE events
- Reloads conversations list on new message
- Auto-cleanup on component unmount

**Empty State:**
- Friendly icon and message
- "No messages yet - start a conversation!" text
- "Browse Creators" button → navigates to /feed
- Centered layout with shadow card

**Timestamp Formatting:**
- Just now (< 1 min)
- 5m ago (< 1 hour)
- 2h ago (< 24 hours)
- 3d ago (< 7 days)
- "Jan 15" (>= 7 days)

**Ephemeral Messaging Hint:**
- Shows "💡 Messages disappear after 24 hours" below list
- Only visible when conversations exist
