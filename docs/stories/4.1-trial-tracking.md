## Technical Notes

```sql
ALTER TABLE users
ADD COLUMN trial_started_at TIMESTAMPTZ,
ADD COLUMN trial_ends_at TIMESTAMPTZ,
ADD COLUMN subscription_status TEXT DEFAULT 'trial'
  CHECK (subscription_status IN ('trial', 'paid', 'free_feetfans', 'expired', 'consumer'));

-- Cron job (node-cron)
cron.schedule('0 0 * * *', async () => {
  await supabase
    .from('users')
    .update({ subscription_status: 'expired' })
    .lt('trial_ends_at', new Date())
    .eq('subscription_status', 'trial');
});
```

## Dev Agent Record

### Implementation Summary
Created trial tracking system with automatic initialization on age verification, daily expiration cron job, and UI countdown component.

### Technical Decisions
- **Direct users table columns:** trial_started_at, trial_ends_at, subscription_status added to users table
- **Trigger-based initialization:** Auto-set trial when age_verified changes to true
- **7-day trial period:** Standard for all creators
- **Consumer exemption:** Consumers marked as 'consumer' status (no trial)
- **Daily expiration check:** Cron job runs daily to expire trials
- **Index on trial_ends_at:** Optimize cron job queries
- **Subscription status enum:** trial, active, free_feetfans, expired, cancelled, consumer

### Features Implemented
**Database:**
- Added trial_started_at, trial_ends_at, subscription_status columns
- Trigger initialize_creator_trial() on age verification
- Function expire_trials() returns expired users
- Index for efficient trial expiration queries

**Cron Job:**
- GET /api/cron/expire-trials endpoint
- Bearer token auth with CRON_SECRET
- Calls expire_trials() SQL function
- Logs expired users with details
- Returns count and list of expired users

**UI Component:**
- TrialCountdown component shows days remaining
- Updates every hour
- Shows "Trial Expired" for expired trials
- Blue badge for active trials, red for expired
